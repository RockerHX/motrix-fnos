# 飞牛版 Motrix UI 设计文档（Stitch 提示词版）

> 用途：给 Google Stitch 生成飞牛版 Motrix 的 UI 原型。  
> 结构：**前半部分是中文设计说明**，**后半部分统一放 Stitch 英文提示词**。  
> 说明：每个页面都标了对应的提示词编号，方便你直接对照。

## 1. 设计目标

这个 UI 是给 **飞牛 OS** 上运行的下载工具使用，不是 Web 站点，也不是手机 App。

目标：

- 像系统原生应用一样稳定、直接、清晰
- 支持中文界面
- 适合长期下载、任务管理和后台运行
- 具备 Motrix 类似的高密度任务管理体验
- 风格克制、专业、现代，避免花哨

## 2. 全局视觉规范

### 2.1 产品定位

- 飞牛 OS 桌面应用
- 下载管理器
- 重点关注任务管理、速度、清晰度和稳定性
- 不是浏览器网页应用，也不是手机应用

### 2.2 视觉风格

- 深色模式优先
- 干净、现代、接近原生桌面应用
- 使用柔和绿色强调下载动作和进度
- 对比度要足够，但不要刺眼
- 圆角建议 10–14px
- 信息密度可以较高，但必须保持可读性
- 阴影克制，边框细腻，不使用夸张玻璃拟态

### 2.3 颜色建议

- 背景色：近黑色 / 深炭灰
- 卡片和面板：比背景稍浅的深灰
- 强调色：下载绿色
- 危险色：红色
- 警告色：琥珀黄
- 成功色：绿色
- 文字色：高对比白色 / 灰阶

### 2.4 字体建议

- 中文界面：苹方 / 系统无衬线字体
- 英文和数字：清晰的无衬线字体
- 标题使用中等字重
- 正文使用常规字重

### 2.5 布局建议

- 左侧侧边栏展示任务分类
- 顶部工具栏放主要操作
- 中间内容区展示任务表格
- 底部状态栏展示全局速度和连接信息
- 右侧抽屉或弹窗展示任务详情和高级设置

### 2.6 组件建议

- 紧凑型任务表格
- 进度条同时展示百分比、速度和剩余时间
- 圆角主按钮
- 不可用操作使用幽灵按钮或禁用状态
- 空状态插画
- 状态使用行内徽标
- 右上角搜索框
- 任务行支持右键菜单

## 3. 页面清单

1. 主窗口 / 全部任务页
2. 新建任务弹窗
3. 任务详情侧栏
4. 设置页
5. 首次空状态页
6. 关于页

> 说明：下载中、已完成、做种、暂停、错误等，建议作为主窗口左侧分类和筛选状态，而不是单独拆成完全不同的页面。

## 4. 页面设计说明

### 4.1 主窗口 / 全部任务页

**用途：** 用户进入应用后的核心首页，负责查看和管理所有任务。  
**对应提示词：** `P1`

#### 页面布局

- 左侧固定侧边栏，宽度约 220–260px
- 顶部工具栏，包含新增任务、开始、暂停、删除、搜索
- 中间为任务列表区域
- 底部状态栏显示总下载速度、上传速度、连接数、磁盘状态
- 右下角可以放“任务详情”入口或折叠按钮

#### 左侧分类建议

- 全部
- 下载中
- 已完成
- 做种
- 活动
- 暂停
- 错误

每个分类后面显示数量徽标。

#### 任务列表字段建议

- 任务名称
- 状态
- 进度
- 已下载 / 总大小
- 下载速度
- 剩余时间
- 保存路径
- 操作按钮

#### 交互细节

- 单击选中任务
- 双击打开任务详情
- 选中任务后顶部按钮可用
- 未选中时按钮置灰
- 鼠标悬停显示更多操作
- 右键打开上下文菜单

#### 空列表状态

- 显示插画
- 文案：无任务
- 提供“添加任务”主按钮
- 提示支持链接粘贴

### 4.2 新建任务弹窗

**用途：** 用户快速新增下载任务，是高频操作入口。  
**对应提示词：** `P2`

#### 页面布局

- 中央弹窗
- 顶部标题 + 关闭按钮
- 支持多种输入方式的标签页或分段控制
- 底部固定操作按钮：取消、开始下载

#### 推荐输入模式

- URL 下载
- 批量 URL
- 种子文件（后期）
- 磁力链接（后期）

#### 表单字段建议（最小版）

- 下载链接
- 文件名（可编辑）
- 保存路径
- 开始方式：立即 / 暂停后添加
- 备注（可选）

#### 高级字段（可折叠）

- 分类
- 线程数 / 分段数
- 限速
- 用户名 / 密码（如需）
- 代理（后期）

#### 交互细节

- 粘贴链接自动识别
- 自动读取文件名
- 保存路径支持浏览选择
- 非法链接即时校验
- 开始按钮仅在输入合法时可用

### 4.3 任务详情侧栏

**用途：** 查看单个任务的详细信息、文件信息和控制操作。  
**对应提示词：** `P3`

#### 页面布局

- 从右侧滑出的 drawer
- 顶部显示任务名称、状态和操作按钮
- 中间分区展示任务详情
- 底部展示日志、链接信息或更多控制项

#### 建议内容

- 基本信息：状态、进度、大小、速度、ETA、保存路径
- 连接信息：线程数、来源数、重试次数
- 文件信息：文件名、目录、已下载文件
- 控制信息：开始、暂停、继续、删除
- 历史信息：创建时间、完成时间、失败原因

#### 可选标签页

- 概览
- 文件
- 连接
- 日志
- 详情

#### 交互细节

- 支持复制下载链接
- 支持打开保存目录
- 支持单个文件重命名（后期）
- 支持查看失败原因

### 4.4 设置页

**用途：** 配置下载默认行为、界面偏好和系统集成。  
**对应提示词：** `P4`

#### 页面布局

- 左侧设置分类
- 右侧表单内容
- 底部保存按钮或自动保存提示

#### 推荐分类

- 常规
- 下载
- 任务
- 界面
- 通知
- 高级
- 关于

#### 常规设置

- 启动时行为
- 关闭窗口行为
- 是否后台运行
- 是否开机自启

#### 下载设置

- 默认保存路径
- 同时下载数
- 每任务连接数
- 全局限速
- 任务完成后动作

#### 界面设置

- 深色 / 浅色
- 语言
- 字体大小
- 任务列表密度

#### 高级设置

- Aria2 Next 路径
- RPC 端口
- 日志级别
- 数据库位置

#### 交互细节

- 修改后可立即保存
- 对危险项进行确认
- 重置为默认值

### 4.5 首次空状态页

**用途：** 应用第一次打开、没有任务时的引导页。  
**对应提示词：** `P5`

#### 页面布局

- 中央大插画
- 简短标题
- 说明文字
- 主按钮：添加任务
- 次按钮：查看设置

#### 文案方向

- 无任务
- 现在添加第一个下载任务
- 支持 HTTP / HTTPS 链接

#### 交互细节

- 主按钮高可见
- 提示可直接粘贴链接
- 空态不要显得像报错页

### 4.6 关于页

**用途：** 展示版本、引擎信息和帮助入口。  
**对应提示词：** `P6`

#### 页面布局

- 应用 Logo
- 应用名称和版本号
- Aria2 Next 版本信息
- 构建信息
- 许可证信息
- 检查更新按钮

#### 交互细节

- 显示当前运行环境
- 提供日志导出入口（后期）
- 提供问题反馈入口

## 5. Stitch 提示词库

> 下面这一部分可以直接复制给 Stitch。  
> 每个 prompt 对应上面的页面说明编号。

### P1 主窗口 / 全部任务页

```text
请设计一个运行在飞牛 OS 上的桌面下载管理器主界面，应用名称为“Motrix FNOS”。整体使用深色模式，风格接近系统原生桌面应用，而不是网页后台。

界面结构：左侧固定分类栏、顶部操作工具栏、中间任务列表表格、底部全局状态栏。
视觉风格：干净、现代、专业、克制；使用绿色作为下载动作和进度的强调色；信息密度较高但保持可读；少量阴影、细边框、圆角卡片，不要使用夸张的玻璃拟态效果。
具体内容：左侧分类包含“全部、下载中、已完成、做种、活动、暂停、错误”，每个分类右侧显示任务数量徽标；顶部操作包含“添加任务、开始、暂停、删除”，右侧放搜索框；中间任务表格列包含“任务名称、状态、进度、已下载/总大小、速度、剩余时间、保存路径、操作”；没有任务时显示空状态插画和“无任务”提示；需要表现鼠标悬停、选中行、工具栏按钮禁用状态；底部状态栏显示全局下载速度和上传速度。
```

### P2 新建任务弹窗

```text
请设计一个“新建下载任务”的居中弹窗，用于桌面下载管理器。保持与主界面一致的飞牛 OS 深色风格，并使用绿色作为主要操作按钮的强调色。

界面结构：居中弹窗，顶部有清晰标题和关闭按钮，中间是输入区域，底部是操作按钮。
视觉风格：高效、简洁、可靠、精致，像原生桌面应用，不要像网页表单。
具体内容：顶部提供输入类型切换，包含“URL 下载、批量 URL、种子文件（后期）、磁力链接（后期）”；主要字段包含“下载链接、文件名、保存路径、开始方式、备注”；开始方式包含“立即开始”和“添加后暂停”；提供一个可折叠的“高级设置”区域，包含“分类、连接数、限速、用户名/密码、代理（后期）”；底部按钮为“取消”和“开始下载”；当链接无效时在输入框下方显示即时校验提示；输入无效时“开始下载”按钮置灰不可点击。
```

### P3 任务详情侧栏

```text
请设计一个从右侧滑出的“任务详情”侧边栏，用于桌面下载管理器中查看单个任务的详细信息。保持与主界面一致的深色主题和绿色强调色。

界面结构：右侧抽屉面板，顶部显示任务标题、状态徽标和常用操作按钮，中间使用标签页展示不同信息，底部放辅助操作或状态信息。
视觉风格：信息清晰、冷静、精确、可读性强，像原生桌面应用。
具体内容：展示任务名称、状态徽标、进度、下载速度、剩余时间、已下载大小、总大小、保存路径、创建时间、完成时间、重试次数、来源数；标签页包含“概览、文件、连接、日志、详情”；操作按钮包含“暂停、继续、打开目录、复制链接、删除”；“文件”标签页展示紧凑的文件列表；“日志”标签页展示按时间排序的日志时间线。
```

### P4 设置页

```text
请设计一个桌面下载管理器的“设置”页面，使用左侧设置分类导航和右侧内容面板。整体保持飞牛 OS 原生应用的深色风格。

界面结构：两栏布局，左侧是设置分类，右侧是表单内容区域。
视觉风格：结构清晰、冷静、专业、偏工具型，重点是易理解和易配置。
具体内容：左侧分类包含“常规、下载、任务、界面、通知、高级、关于”；“常规”包含启动时行为、关闭窗口行为、是否后台运行、是否开机自启；“下载”包含默认保存路径、同时下载数、每任务连接数、全局限速、任务完成后动作；“界面”包含深色/浅色主题、语言、任务列表密度；“高级”包含 Aria2 Next 路径、RPC 端口、日志级别、数据库位置；表单控件使用开关、下拉框、输入框、路径选择器；底部提供“保存”和“恢复默认设置”操作区。
```

### P5 首次空状态页

```text
请设计一个桌面下载管理器的首次空状态页面，使用深色模式。

界面结构：中央插画、标题、说明文字、主按钮、次按钮。
视觉风格：友好、干净、安心、简洁，保持原生桌面应用感觉。
具体内容：中央显示一个下载文件夹插画；标题为“暂无任务”；说明文字提示用户可以添加 HTTP 或 HTTPS 链接开始下载；主按钮为“添加任务”；次按钮为“打开设置”；整体保持与应用其他页面一致的绿色强调色、圆角和深色背景；空状态不要像错误页，要像正常引导页。
```

### P6 关于页

```text
请设计一个飞牛 OS 桌面下载管理器的“关于”页面。

界面结构：居中或卡片式布局，包含应用图标、应用名称、版本信息、构建信息和帮助入口。
视觉风格：简洁、可信、精致、像原生桌面应用，保持深色主题。
具体内容：展示应用 Logo、应用名称“Motrix FNOS”、应用版本号、Aria2 Next 引擎版本、构建时间、许可证信息；提供“检查更新”按钮；提供“问题反馈”和“使用文档”链接；页面整体要安静、清晰，与其他深色页面风格一致。
```

## 6. 设计一致性要求

所有页面都要遵守以下规则：

- 同一套暗色主题
- 同一套绿色强调色
- 同一套圆角和间距系统
- 同一套图标风格
- 同一套中文文案语气
- 同一套任务状态颜色

### 6.1 状态颜色建议

- 下载中：绿色
- 暂停：灰蓝
- 已完成：绿色或浅蓝
- 错误：红色
- 排队：黄色

### 6.2 任务状态展示方式

- 使用徽标 + 颜色
- 不要只靠文字
- 进度条必须清晰可见
- 速度和 ETA 放在同一信息组中

## 7. 推荐的 Stitch 使用方式

1. 先用 **P1 主窗口** 生成整体视觉基调
2. 再用 **P2 新建任务弹窗**、**P3 任务详情侧栏**、**P4 设置页** 细化
3. 如果需要空状态，再用 **P5**
4. 关于页用 **P6** 收尾
5. 每次只改一个重点，避免风格跑偏
